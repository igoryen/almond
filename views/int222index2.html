About HTML and Reset

HTML5 is becoming very popular in web development and the reasons are following: accessibility (sites are accessible), supports video and audio content, helps create clean code, better interactions with the user, supports other browsers including legacy browsers, creates mobile sites, and is the language of the future.

Why do we use reset? We use the short of CSS rules to reset the styling of all HTML elements to a consistent baseline. Every browser has its own default "user agent" stylesheet, that it uses to make unstyled websites appear more legible. Using a CSS Reset, we can force every browser to have all its styles reset to null, thus avoiding cross-browser differences as much as possible.
